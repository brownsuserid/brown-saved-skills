# Outreach Plan Template

Full template for the 8-section outreach plan written into the Deal's Outreach Plan field in Airtable. Each section header uses `##`. When updating an existing plan, append new entries within each section — do not replace existing content.

---

## 1. Deal Snapshot

Current state summary. Update this section on each planning run.

```
## Deal Snapshot
Last Updated: <date>

Deal: <deal name>
Status: <deal status>
Pipeline Stage: <stage>

Primary Contact: <name> | <title>
  Email: <email>
  Phone: <phone>
  LinkedIn: <linkedin_url>

Organization: <org name>
  Industry: <industry>
  Size: <employee count or range>
  Location: <city, state/country>

Campaign: <campaign name> (or "None")

Key Qualification Signals:
- <signal 1>
- <signal 2>
```

## 3. Outreach State

Tracks where the deal is in the sequence. Update on each run.

```
## Outreach State
Last Updated: <date>

Touch Number: <n> of <sequence length>
Last Action: <action type> on <date>
Last Response: <summary of response> on <date> (or "None")
Response Classification: <Positive | Objection | Confusion | Hostile | OOO | None>
Days Since Last Contact: <n>

Sequence Position: <e.g., "Completed touch 2 of 5 — awaiting reply">
```

**Response classification guide:**
- Positive — interest, questions, agreed to meeting
- Objection — pushback on value, timing, or relevance
- Confusion — unclear why they were contacted
- Hostile — anger, demands to stop
- OOO — out-of-office auto-reply
- None — no reply received

## 4. Forward Plan

Planned next actions. Each entry maps to a task that will be created in Step 2. Append new forward plans with a date stamp; preserve prior entries.

```

[<date>]

  Type: <Send Email | LinkedIn Message | Phone Call | Research | Human Communication>
  Channel: <channel>
  Timing: <due date or relative timing, e.g., "2 days">
  Content Strategy: <what angle or hook to use>
  Authority Level: <A | B | C | D>
  Task Name: <task name as it will appear in Airtable>

Action 2: (if applicable)
  ...

Action 3: (if applicable)
  ...
```

---

## 5. Conditional Forward Planning

If/then scenarios based on likely responses to the Forward Plan. Write once per plan cycle; append new scenarios with date stamps.

```
## Conditional Forward Planning
[<date>]

If positive response (interest / questions / agree to meeting):
→ Advance qualification; propose meeting; compress timing to within 48 hours

If objection (<specific objection type>):
→ <specific handling — acknowledge, address root cause, provide evidence>

If no response after this touch:
→ <next touch in sequence, or disengagement protocol if sequence is complete>

If new stakeholder identified:
→ Create Research task on new contact; add to stakeholder map; Level C approval before outreach

If OOO:
→ Pause all outreach; resume after <return date>; adjust all task due dates
```

## 6. Plan Deviations Log

Any deviation from the campaign plan must be recorded here. Append entries chronologically; never remove prior entries.

```
## Plan Deviations Log

| Date | Original Plan | Deviation | Reason |
|------|---------------|-----------|--------|
| <date> | <what the campaign plan called for> | <what was done instead> | <why> |
```

If no deviations have occurred, write: `No deviations from campaign plan.`

## 8. HITL Communication Log

Log of all human-in-the-loop interactions — questions asked, recommendations made, approvals given. Append entries chronologically; never remove prior entries.

```
## HITL Communication Log

| Date | Type | Message | Response | Resolution |
|------|------|---------|----------|------------|
| <date> | <question|recommendation|authority_request|blocker> | <what was asked or communicated> | <owner's response> | <how it was resolved> |
```

Types:
- **question** — asked owner for clarification or information
- **recommendation** — proposed an action for owner awareness
- **authority_request** — requested approval before executing a Level C task
- **blocker** — flagged an issue that prevents progress

If no HITL interactions have occurred, write: `No HITL interactions recorded.`
