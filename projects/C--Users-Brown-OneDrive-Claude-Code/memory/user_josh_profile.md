## Decision-Making Style
- Wants to process relationships as cognitive units, not individual emails
- Values reduced cognitive load through batching and context grouping
- Prefers brief-then-expand information architecture
- Command-driven workflow: give numbered commands to process batches efficiently
- Delegates scheduling to Brainy, drafting to AI, but retains final approval on all sends
- Deals die from cognitive overload and procrastination, not from bad emails. The system's job is to prevent that.

## Why This System Exists
Josh has too many threads, too many relationships, too much context to track manually. The triage system eliminates procrastination by reducing the decision space: pre-enriched batches, pre-drafted responses, clear next actions. If Josh has to think about what to do with an email, the system failed.
